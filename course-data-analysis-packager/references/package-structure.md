# Package Structure

Prefer this structure for a new course data analysis package:

```text
project_name/
  README.md
  data/
    raw/
    processed/
  docs/
    data_dictionary.md
    report_template.md
    ppt_slide_outline.md
  scripts/
  notebooks/
  output/
    tables/
    figures/
```

## File Responsibilities

- `README.md`: topic, research question, data source, quick run commands, output list, limitations.
- `data/raw/`: original files, never overwritten.
- `data/processed/`: cleaned intermediate files generated by scripts.
- `docs/data_dictionary.md`: field names, type, meaning, missing-value handling, derived fields.
- `docs/report_template.md`: report skeleton with placeholders for figures and findings.
- `docs/ppt_slide_outline.md`: slide titles, message, evidence, figure/table mapping.
- `scripts/`: reproducible extraction, cleaning, analysis, and visualization scripts.
- `output/tables/`: CSV/Excel tables that can be cited.
- `output/figures/`: PNG/SVG charts sized for PPT.

## Naming Rules

- Use lowercase English folder names when possible.
- Use date or version suffixes only when they help preserve handoff history.
- Avoid spaces in script names.
- Keep raw and processed files separate.
