# Deliverable QA

## Reproducibility

- README includes exact run commands.
- Scripts run from the project root.
- Output paths are deterministic.
- Required packages are listed or obvious from imports.
- Raw data is not modified by scripts.

## Data Quality

- Data dictionary covers all columns used in the report.
- Missing values are handled explicitly.
- Duplicate rows are checked when relevant.
- Sensitive or personal fields are removed from public outputs.

## Report Quality

- Research question is stated early.
- Charts support the main claims.
- Limitations are visible.
- No unsupported causal claims.
- Report and PPT outline use the same metric names.

## Presentation Readiness

- Figures are readable in slides.
- Tables are not too wide.
- Slide outline has one claim per slide.
- Every slide points to a figure, table, or source note when possible.
